VEX & CSAF Advisory Generation Runbook

Step-by-step guide for triaging vulnerabilities, classifying each CVE with a VEX status, generating the VEX document, and producing a CSAF advisory with CVSS scores and mitigation steps. CycloneDX JSON · CERT-In v2.0.

1

Overview & Document Relationship

Two distinct documents are produced in this phase. Both are confidential and must never appear in the public SBOM or client delivery without explicit authorisation.

DocumentPurposeAudienceFormat
VEX
vex.json		 Declares the exploitability status of every CVE found in the SBOM Client security team CycloneDX VEX JSON
CSAF
csaf.json		 Formal security advisory with CVSS scores, affected versions, and mitigations Client & internal stakeholders CSAF 2.0 JSON
ℹ️ Flow: Trivy scan → enriched SBOM → triage CVEs here → VEX document → CSAF advisory → embed VEX ref in private SBOM.

VEX Status Definitions

Every CVE in the SBOM must be assigned exactly one of the four VEX statuses below before the document is generated. Make triage decisions based on your knowledge of the deployment environment and component usage.

Not Affected

The vulnerability exists in the component but cannot be exploited in this deployment context. Provide a justification (e.g., vulnerable code path is unreachable).

Affected

The vulnerability is present and exploitable in the current build. Immediate remediation or compensating control is required. Must appear in the CSAF advisory.

Fixed

The vulnerability has been remediated — either by upgrading the component or applying a patch. Provide the fixed version and remediation date.

Under Investigation

Exploitability is not yet determined. Assign a review deadline. This status must be resolved before final client delivery.

🚫 Never leave a CVE without a status. An unclassified vulnerability is a compliance failure under CERT-In Technical Guidelines v2.0.

Generate the VEX Document

Save the script below as sbom_toolsuite/vex_csaf_generator.py, then run it from the project root. It reads triage.json and the enriched SBOM to produce sbom_output/vex.json. 

python sbom_toolsuite/vex_csaf_generator.py
# ── sbom_toolsuite/vex_csaf_generator.py ──────────────────────
import json, datetime, os, uuid

ENRICHED = 'sbom_output/sbom_enriched.json'
TRIAGE   = 'sbom_toolsuite/triage.json'
CONFIG   = 'sbom_toolsuite/config.json'
VEX_OUT  = 'sbom_output/vex.json'
CSAF_OUT = 'sbom_output/csaf.json'
os.makedirs('sbom_output', exist_ok=True)

with open(ENRICHED) as f: sbom    = json.load(f)
with open(TRIAGE)   as f: triage  = json.load(f)
with open(CONFIG)   as f: config  = json.load(f)

now = datetime.datetime.utcnow().isoformat() + "Z"

# ── Build CVE → SBOM vulnerability lookup ─────────────────────
sbom_vulns = {v['id']: v for v in sbom.get('vulnerabilities', [])}

# ── VEX Document ──────────────────────────────────────────────
vex = {
    "bomFormat":   "CycloneDX",
    "specVersion": "1.5",
    "version":     1,
    "serialNumber": f"urn:uuid:{uuid.uuid4()}",
    "metadata": {
        "timestamp": now,
        "authors": [{"name": config["author"]}],
        "component": {"name": config["organization"], "type": "application"}
    },
    "vulnerabilities": []
}

for t in triage:
    base = sbom_vulns.get(t['cve_id'], {})
    ratings = base.get('ratings', [{}])

    entry = {
        "id": t['cve_id'],
        "source": {"name": "NVD", "url": f"https://nvd.nist.gov/vuln/detail/{t['cve_id']}"},
        "ratings": ratings,
        "affects": [{
            "ref": t['component'],
            "versions": [{"version": t['version'], "status": t['status']}]
        }],
        "analysis": {
            "state":         t['status'],
            "justification": t['justification'],
            "response":      ["will_not_fix"] if t['status'] == 'not_affected'
                             else ["update"] if t['status'] == 'fixed'
                             else ["workaround_available"] if t['status'] == 'affected'
                             else []
        }
    }
    vex["vulnerabilities"].append(entry)

with open(VEX_OUT, 'w') as f:
    json.dump(vex, f, indent=2)
print(f"[OK] vex.json written — {len(triage)} vulnerabilities classified.")
✅ Expected: [OK] vex.json written — N vulnerabilities classified.

Generate the CSAF Advisory

The CSAF block is generated by the same vex_csaf_generator.py script. Append the following section to the script before the final print statement. It writes sbom_output/csaf.json — a CSAF 2.0 compliant advisory. 

# ── Append to vex_csaf_generator.py (after VEX write) ────────

# Filter only 'affected' entries for the advisory
affected = [t for t in triage if t['status'] == 'affected']

csaf = {
    "document": {
        "csaf_version": "2.0",
        "title":        f"Security Advisory — {config['organization']}",
        "publisher": {
            "name":      config["author"],
            "namespace": "https://antigravity.ai/security",
            "category":  "vendor"
        },
        "tracking": {
            "id":                f"AA-{datetime.datetime.utcnow().strftime('%Y%m%d')}-001",
            "status":            "final",
            "version":           "1",
            "initial_release_date": now,
            "current_release_date": now,
            "revision_history": [{
                "date":    now,
                "number": "1",
                "summary":"Initial release"
            }]
        },
        "distribution": {"tlp": {"label": "AMBER"}},
        "references": [{
            "url":      "https://nvd.nist.gov",
            "summary": "National Vulnerability Database"
        }]
    },
    "product_tree": {
        "branches": [{
            "category": "product_family",
            "name":     config["organization"],
            "branches": [{
                "category": "product_version",
                "name":     t["component"] + "@" + t["version"],
                "product":  {
                    "name":       t["component"],
                    "product_id": t["component"]
                }
            } for t in affected]
        }]
    },
    "vulnerabilities": [{
        "cve": t["cve_id"],
        "title": f"Affected: {t['component']} {t['version']}",
        "notes": [{
            "category": "description",
            "text": t["justification"]
        }],
        "product_status": {
            "known_affected":    [t["component"]],
            "fixed":             [t["component"]] if t.get("fixed_version") else []
        },
        "remediations": [{
            "category":   "vendor_fix",
            "details":    f"Upgrade to {t['component']} {t['fixed_version']}"
                          if t.get("fixed_version") else "Apply compensating control — see justification",
            "product_ids": [t["component"]]
        }],
        "scores": sbom_vulns.get(t['cve_id'], {}).get('ratings', [])
    } for t in affected]
}

with open(CSAF_OUT, 'w') as f:
    json.dump(csaf, f, indent=2)
print(f"[OK] csaf.json written — {len(affected)} affected vulnerabilities documented.")
✅ Expected: [OK] csaf.json written — N affected vulnerabilities documented.
If N is 0, all CVEs are either fixed, not_affected, or under investigation — no CSAF advisory entries are needed.
⚠️ CSAF TLP is set to AMBER by default — meaning it is restricted to the client organisation only and must not be published publicly.

Validate Both Outputs

Run this validation script to confirm both documents are well-formed, all required fields are populated, and no CVE in the triage list is missing from the VEX document. 

python -c "
import json, sys

errors = []

# ── Validate VEX ─────────────────────────────────────────────
with open('sbom_output/vex.json') as f: vex = json.load(f)
VALID_STATES = {'not_affected','affected','fixed','under_investigation'}
for v in vex.get('vulnerabilities', []):
    state = v.get('analysis', {}).get('state', '')
    if state not in VALID_STATES:
        errors.append(f'VEX [{v[\"id\"]}]: invalid state \"{state}\"')
    if not v.get('analysis', {}).get('justification'):
        errors.append(f'VEX [{v[\"id\"]}]: missing justification')

# ── Validate CSAF ─────────────────────────────────────────────
with open('sbom_output/csaf.json') as f: csaf = json.load(f)
doc = csaf.get('document', {})
if doc.get('csaf_version') != '2.0':
    errors.append('CSAF: csaf_version must be \"2.0\"')
if not doc.get('tracking', {}).get('id'):
    errors.append('CSAF: tracking.id is missing')
if doc.get('distribution', {}).get('tlp', {}).get('label') != 'AMBER':
    errors.append('CSAF: TLP label must be AMBER')
for v in csaf.get('vulnerabilities', []):
    if not v.get('remediations'):
        errors.append(f'CSAF [{v.get(\"cve\")}]: missing remediation entry')

if errors:
    print('VALIDATION FAILED:')
    for e in errors: print(f'  !! {e}')
    sys.exit(1)
else:
    vex_count  = len(vex['vulnerabilities'])
    csaf_count = len(csaf['vulnerabilities'])
    print(f'[OK] VEX  validated — {vex_count} entries, all states and justifications present.')
    print(f'[OK] CSAF validated — {csaf_count} affected entries, all remediations present.')
"
✅ Expected: Both [OK] lines printed. Any !! line must be resolved before client delivery.

Completion Checklist

  • All CVEs from sbom_enriched.json listed and reviewed

  • No CVE left without a VEX status (zero unclassified entries)
  • All under_investigation entries have a review deadline in the justification
  • vex_csaf_generator.py executed without errors
  • vex.json written to sbom_output/
  • csaf.json written to sbom_output/
  • Validation script passed — all VEX states valid, all CSAF remediations present
  • CSAF TLP label confirmed as AMBER
  • Export-controlled components cross-checked against affected vulnerability list
  • Both files stored in RBAC-locked location — not in client public delivery folder
