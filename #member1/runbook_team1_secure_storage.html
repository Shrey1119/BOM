Secure Storage Integration Guidelines

Step-by-step guide for classifying, storing, protecting, transmitting, and auditing all SBOM artifacts — keys, documents, advisories, and signatures — in compliance with CERT-In Technical Guidelines v2.0.

Artifact Classification Policy

Every artifact must carry an explicit classification label. Use these definitions consistently across all storage, transmission, and audit records.

LabelDefinitionSharing PermittedEncryption Required
Secret Cryptographic key material. Exposure compromises all signed artifacts. Never. Local system only. Yes — at rest & never transmitted
Confidential Contains vulnerability data, internal component details, or proprietary info. Client only via encrypted channel. Yes — at rest & in transit
TLP:AMBER Restricted to client organisation. Not for further redistribution. Client organisation only. Yes — in transit
Internal Organisation-only records. Not for client delivery. Internal staff with need-to-know. Yes — at rest
Public Non-sensitive. Safe for open distribution. Anyone via HTTPS with integrity checksum. No — but must be checksummed
3

Storage Tiers

Three storage tiers are defined. Each artifact must be placed in exactly one tier. Tier assignment is permanent — files must not be moved to a lower tier.

🔴 Tier 1 — Restricted

For Secret and Confidential artifacts. RBAC-locked directory, access-logged, encrypted at rest. No public read permissions under any circumstance.

🟡 Tier 2 — Internal

For TLP:AMBER and Internal artifacts. Access limited to authenticated staff. Read-only for most roles. Encrypted in transit.

🟢 Tier 3 — Public

For Public artifacts only. Served via HTTPS with a SHA-256 checksum file alongside each artifact. No sensitive data.

Directory Structure

Provision the following folder layout on the scanning environment before running the toolsuite. Each folder maps to a storage tier. 

# Recommended layout — project root: c:\Study\Projects\BOM\
sbom_output\
├── restricted\          ← Tier 1: private_key.pem, sbom_private.json, sbom_enriched.json, vex.json
├── internal\            ← Tier 2: internal_map.json, csaf.json, triage.json
└── public\              ← Tier 3: sbom_public.json, *.sig, *.sha256
# PowerShell — create directory structure
New-Item -ItemType Directory -Force sbom_output\restricted
New-Item -ItemType Directory -Force sbom_output\internal
New-Item -ItemType Directory -Force sbom_output\public
# Python helper — move outputs to correct tier after generation
python -c "
import shutil, os

moves = {
    'sbom_output/sbom_private.json':  'sbom_output/restricted/',
    'sbom_output/sbom_enriched.json': 'sbom_output/restricted/',
    'sbom_output/vex.json':           'sbom_output/restricted/',
    'sbom_output/internal_map.json':  'sbom_output/internal/',
    'sbom_output/csaf.json':          'sbom_output/internal/',
    'sbom_output/sbom_public.json':   'sbom_output/public/',
}

for src, dst in moves.items():
    if os.path.exists(src):
        shutil.move(src, dst)
        print(f'[moved] {src} -> {dst}')
    else:
        print(f'[skip]  {src} not found')
"
⚠️ The sbom_output/ root directory itself must be added to .gitignore. No generated artifacts should ever be committed to version control.

Access Controls

Apply OS-level permissions to each tier directory immediately after creation. Review and confirm permissions after every toolsuite run.

Windows — PowerShell (run as Administrator)

# ── Tier 1: Restricted — only scanning service account ────────
icacls sbom_output\restricted /inheritance:r
icacls sbom_output\restricted /grant:r "$($env:USERNAME):(OI)(CI)F"

# ── Tier 2: Internal — service account + security lead ────────
icacls sbom_output\internal /inheritance:r
icacls sbom_output\internal /grant:r "$($env:USERNAME):(OI)(CI)F"
icacls sbom_output\internal /grant:r "BUILTIN\Administrators:(OI)(CI)R"

# ── Tier 3: Public — read access for delivery group ───────────
icacls sbom_output\public /grant "Everyone:(OI)(CI)R"

Linux / macOS

# Tier 1 — owner only
chmod 700 sbom_output/restricted
chmod 600 sbom_output/restricted/*

# Tier 2 — owner + group read
chmod 750 sbom_output/internal
chmod 640 sbom_output/internal/*

# Tier 3 — world-readable
chmod 755 sbom_output/public
chmod 644 sbom_output/public/*

Encryption at Rest

Tier 1 and Tier 2 artifacts must be encrypted at rest. Choose the method appropriate to your environment.

MethodPlatformTier CoverageSteps
BitLocker Windows Tier 1 & 2 Enable BitLocker on the drive or volume hosting sbom_output\restricted and sbom_output\internal. Escrow recovery key to a secure vault.
LUKS Linux Tier 1 & 2 Mount sbom_output/restricted on a LUKS-encrypted volume. Key stored in HSM or systemd-creds.
File-level AES-256 Any Tier 1 only Encrypt individual Tier 1 files with AES-256-GCM using the script below. Suitable when full-disk encryption is not available. 
# File-level AES-256-GCM encryption (Python) — for Tier 1 files
pip install cryptography

python -c "
import os, sys
from cryptography.hazmat.primitives.ciphers.aead import AESGCM

def encrypt_file(src, passphrase: bytes):
    key  = passphrase[:32].ljust(32, b'0')   # 256-bit key
    aesgcm = AESGCM(key)
    nonce  = os.urandom(12)
    with open(src, 'rb') as f: data = f.read()
    ct = aesgcm.encrypt(nonce, data, None)
    with open(src + '.enc', 'wb') as f: f.write(nonce + ct)
    os.remove(src)
    print(f'[encrypted] {src} -> {src}.enc')

PASSPHRASE = os.environ.get('SBOM_ENC_KEY', '').encode()
if not PASSPHRASE:
    print('ERROR: set SBOM_ENC_KEY env variable first.'); sys.exit(1)

for fname in ['sbom_output/restricted/sbom_private.json',
              'sbom_output/restricted/vex.json',
              'sbom_output/restricted/sbom_enriched.json']:
    if os.path.exists(fname):
        encrypt_file(fname, PASSPHRASE)
"
🚫 The encryption passphrase (SBOM_ENC_KEY) must never be stored in any file, script, or environment config file committed to git. Use a secrets manager, HSM, or OS keychain.

Generate SHA-256 Checksum for Public Artifacts

# PowerShell — generate checksum for public SBOM
$hash = Get-FileHash sbom_output\public\sbom_public.json -Algorithm SHA256
$hash.Hash + "  sbom_public.json" | Out-File sbom_output\public\sbom_public.json.sha256 -Encoding ASCII

Audit Logging

Every access, generation, move, and transmission of Tier 1 and Tier 2 artifacts must be logged. The log is itself a Tier 2 artifact.

Minimum Required Log Fields

FieldExample Value
timestamp2026-06-25T10:32:11Z
actorUsername or service account name
actiongenerated / moved / transmitted / accessed / deleted
artifactsbom_private.json
destinationsbom_output/restricted/ or client-portal
checksumSHA-256 of the file at time of action
 
# audit_log.py — append an audit entry (call from any toolsuite script)
import json, datetime, hashlib, os, getpass

def log_action(artifact_path: str, action: str, destination: str = ""):
    log_file = "sbom_output/internal/audit_log.jsonl"
    checksum = ""
    if os.path.exists(artifact_path):
        with open(artifact_path, "rb") as f:
            checksum = hashlib.sha256(f.read()).hexdigest()
    entry = {
        "timestamp":   datetime.datetime.utcnow().isoformat() + "Z",
        "actor":       getpass.getuser(),
        "action":      action,
        "artifact":    os.path.basename(artifact_path),
        "destination": destination,
        "checksum":    checksum
    }
    os.makedirs(os.path.dirname(log_file), exist_ok=True)
    with open(log_file, "a") as f:
        f.write(json.dumps(entry) + "\n")

# Usage example
log_action("sbom_output/restricted/sbom_private.json", "generated", "sbom_output/restricted/")
log_action("sbom_output/public/sbom_public.json", "transmitted", "client-portal")
ℹ️ audit_log.jsonl uses newline-delimited JSON (one entry per line) for easy parsing and appending without rewriting the whole file.

Integrity Verification

Run this script before any client delivery to confirm that all artifacts match their recorded checksums and that no files have been tampered with since generation. 

python -c "
import hashlib, os, sys

def sha256(path):
    with open(path, 'rb') as f: return hashlib.sha256(f.read()).hexdigest()

checks = [
    ('sbom_output/public/sbom_public.json',
     'sbom_output/public/sbom_public.json.sha256'),
]

errors = []
for artifact, checksum_file in checks:
    if not os.path.exists(artifact):
        errors.append(f'MISSING: {artifact}'); continue
    if not os.path.exists(checksum_file):
        errors.append(f'MISSING CHECKSUM: {checksum_file}'); continue
    with open(checksum_file) as f:
        expected = f.read().split()[0].strip()
    actual = sha256(artifact)
    if actual != expected:
        errors.append(f'TAMPERED: {artifact}')
        errors.append(f'  expected: {expected}')
        errors.append(f'  actual  : {actual}')
    else:
        print(f'[OK] {os.path.basename(artifact)} — checksum verified')

if errors:
    print('\nINTEGRITY CHECK FAILED:')
    for e in errors: print(f'  !! {e}')
    sys.exit(1)
else:
    print('[OK] All artifacts passed integrity verification.')
"
✅ Expected: [OK] All artifacts passed integrity verification.
Any TAMPERED result must be investigated immediately — do not deliver the artifact to the client.

Completion Checklist

  • All artifacts classified with a label from the policy table (Section 2)
  • Three-tier directory structure provisioned: restricted/, internal/, public/
  • OS-level permissions applied to all three tier directories (Section 5)
  • Tier 1 and Tier 2 directories confirmed not world-readable
  • All generated artifacts moved to their correct tier folder
  • sbom_output/ added to .gitignore — confirmed not tracked by git
  • Encryption at rest enabled for Tier 1 (BitLocker / LUKS / file-level AES-256)
  • SHA-256 checksum generated for sbom_public.json and stored alongside it
  • ECDSA signatures (.sig files) present in public/ for all signed artifacts
  • Transmission channel confirmed for each artifact (HTTPS / encrypted email / SFTP)
  • private_key.pem confirmed: local only, not in any transmission log
  • All artifact movements logged in audit_log.jsonl
  • Integrity verification script passed before client delivery
  • Build archive created in sbom_output/archive/ with timestamp
  • CERT-In 3-year retention policy confirmed for all Confidential artifacts
