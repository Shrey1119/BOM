Internal Mapping Compilation Runbook

Step-by-step guide for building and maintaining the organization's internal component map — tracking active components, their lifecycle status, ownership, and compliance flags across the current build. CycloneDX JSON · CERT-In v2.0.

1

Overview & Purpose

The internal mapping is a structured, version-controlled record that your organization maintains independently of the client-facing SBOM. It answers the question: "What exactly is in this build, and do we fully own and understand it?"

InputOutputOwner
Enriched SBOM from Team Member 3 (sbom_enriched.json) Internal component map (internal_map.json) Team Member 1
sbom_toolsuite/config.json (criticality / origin overrides) Usage & risk classification per component Team Member 1
VEX document (vex.json) Vulnerability exposure status cross-referenced in the map Team Member 1
ℹ️ The internal map is never shared with the client. It is an internal governance artifact for audit, legal, and security review only.

Collect & Inspect SBOM Data

Load the enriched SBOM and do a quick field-coverage check before building the map. This confirms Team Member 3's enrichment ran successfully. 

# inspect_sbom.py — run from project root
python -c "
import json, sys

with open('sbom_output/sbom_enriched.json') as f:
    sbom = json.load(f)

components = sbom.get('components', [])
print(f'Total components : {len(components)}')

REQUIRED = ['name', 'version', 'purl', 'licenses', 'hashes']
missing  = {k: 0 for k in REQUIRED}

for c in components:
    for field in REQUIRED:
        if not c.get(field):
            missing[field] += 1

print('\nMissing field counts:')
for field, count in missing.items():
    status = '[OK]' if count == 0 else '[!!]'
    print(f'  {status}  {field}: {count} missing')
"
⚠️ If any critical field shows missing count > 0, notify Team Member 3 to re-run the enricher before proceeding.
5

Build the Internal Component Map

The script below reads the enriched SBOM, merges config overrides, cross-references the VEX document, and writes sbom_output/internal_map.json. 

# build_internal_map.py — save to sbom_toolsuite/ then run from project root
python sbom_toolsuite/build_internal_map.py
# ── sbom_toolsuite/build_internal_map.py ──────────────────────
import json, datetime, os

# Paths
ENRICHED = 'sbom_output/sbom_enriched.json'
VEX      = 'sbom_output/vex.json'
CONFIG   = 'sbom_toolsuite/config.json'
OUTPUT   = 'sbom_output/internal_map.json'

os.makedirs('sbom_output', exist_ok=True)

with open(ENRICHED) as f: sbom   = json.load(f)
with open(CONFIG)   as f: config = json.load(f)

# Load VEX vulnerability statuses (keyed by CVE id)
vex_status = {}
if os.path.exists(VEX):
    with open(VEX) as f: vex = json.load(f)
    for v in vex.get('vulnerabilities', []):
        vex_status[v['id']] = v.get('analysis', {}).get('state', 'unknown')

components = sbom.get('components', [])
crit_map   = config.get('criticality_overrides', {})
origin_map = config.get('origin_overrides', {})
restr_map  = config.get('usage_restrictions_overrides', {})
comments   = config.get('comments', {})

internal_map = {
    "generated_at": datetime.datetime.utcnow().isoformat() + "Z",
    "author": config.get("author"),
    "organization": config.get("organization"),
    "total_components": len(components),
    "components": []
}

for c in components:
    name = c.get('name', '').lower()
    props = {p['name']: p.get('value') for p in c.get('properties', [])}

    # Resolve CVE statuses for this component
    vulns = sbom.get('vulnerabilities', [])
    comp_cves = [
        {"id": v["id"], "severity": v.get("ratings",[{}])[0].get("severity","unknown"),
         "vex_status": vex_status.get(v["id"], "not_evaluated")}
        for v in vulns
        if any(a.get("ref") == c.get("bom-ref") for a in v.get("affects",[]))
    ]

    entry = {
        "name":               c.get('name'),
        "version":            c.get('version'),
        "purl":               c.get('purl'),
        "type":               c.get('type', 'library'),
        "supplier":           c.get('supplier', {}).get('name', 'unknown'),
        "license":            ', '.join(l.get('license',{}).get('id','unknown') for l in c.get('licenses',[])),
        "criticality":        crit_map.get(name, config.get('default_criticality', 'medium')),
        "origin":             origin_map.get(name, config.get('default_origin', 'open-source')),
        "usage_restrictions": restr_map.get(name, config.get('default_usage_restrictions', 'None')),
        "eol_date":           props.get('eol_date', 'unknown'),
        "release_date":       props.get('release_date', 'unknown'),
        "internal_comment":   comments.get(name, ''),
        "cve_exposure":       comp_cves,
        "active_in_build":    True
    }
    internal_map["components"].append(entry)

with open(OUTPUT, 'w') as f:
    json.dump(internal_map, f, indent=2)

print(f'[OK] internal_map.json written — {len(components)} components mapped.')
✅ Expected: [OK] internal_map.json written — N components mapped.

Classify & Tag Components

After the map is generated, apply the following classification rules manually or via the config overrides. Every component must have a confirmed criticality and origin tag.

TagRuleAction Required
High Criticality Listed in criticality_overrides as high, or has active CVE with severity CRITICAL/HIGH Flag for immediate review; include in VEX document
Export Controlled Component name matches usage_restrictions_overrides containing "EAR" or "export" Legal review required before distribution to client
EOL Within 6 Months eol_date is within 180 days of today Notify client; recommend upgrade path in advisory
Proprietary Origin Component listed in origin_overrides as proprietary Strip from public SBOM; retain only in private SBOM
Low / Clean No CVEs, open-source, EOL > 1 year away No action — document as compliant
ℹ️ Add or update overrides in config.json to persist classification decisions permanently across future scans.

Generate Summary Report

Run this script to print a human-readable summary of the internal map — useful for stakeholder reviews and audit records. 

python -c "
import json
from collections import Counter

with open('sbom_output/internal_map.json') as f:
    m = json.load(f)

comps = m['components']
crit  = Counter(c['criticality'] for c in comps)
orig  = Counter(c['origin']      for c in comps)
vuln_count = sum(1 for c in comps if c['cve_exposure'])
export_ctrl = [c['name'] for c in comps if 'export' in c.get('usage_restrictions','').lower() or 'EAR' in c.get('usage_restrictions','')]

print(f'=== INTERNAL MAP SUMMARY ===')
print(f'Generated   : {m[\"generated_at\"]}')
print(f'Organization: {m[\"organization\"]}')
print(f'Total       : {m[\"total_components\"]} components')
print(f'\nCriticality breakdown:')
for k, v in crit.items():   print(f'  {k:<12}: {v}')
print(f'\nOrigin breakdown:')
for k, v in orig.items():   print(f'  {k:<12}: {v}')
print(f'\nComponents with CVE exposure : {vuln_count}')
print(f'Export-controlled components : {len(export_ctrl)}')
if export_ctrl:
    for name in export_ctrl: print(f'  - {name}')
"
✅ Expected: A printed summary table. Review export-controlled and high criticality entries with the legal/security team before client delivery.

Validate & Store the Internal Map

Before archiving, confirm the map is complete and stored in the correct location with proper access controls. 

# Quick validation — ensure every component has the 5 required fields
python -c "
import json, sys
REQUIRED = ['name','version','purl','criticality','license']
with open('sbom_output/internal_map.json') as f:
    m = json.load(f)
errors = []
for i, c in enumerate(m['components']):
    for field in REQUIRED:
        if not c.get(field):
            errors.append(f'Component [{i}] {c.get(\"name\",\"?\")} missing: {field}')
if errors:
    print('VALIDATION FAILED:')
    for e in errors: print(f'  {e}')
    sys.exit(1)
else:
    print(f'[OK] All {len(m[\"components\"])} components pass internal map validation.')
"
