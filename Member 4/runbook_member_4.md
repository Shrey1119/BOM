# Team Member 4 Runbook: Syft & Compliance Pipeline Integration

This runbook outlines the automated pipeline workflow for generating a Software Bill of Materials (SBOM) using **Syft** (deep layer-by-layer dependency cataloger), **Grype** (vulnerability mappings), and a Python post-processing enricher to ensure compliance with the client's **21 mandatory attributes**.

---

## Roles & Responsibilities
As the **Syft Pipeline Specialist**, you are responsible for:
1. Running [setup_tools.py](file:///d:/College%20Work/Internship/BOM/Member%204/setup_tools.py) to download and extract portable Windows binaries of Syft and Grype into [Member 4/bin/](file:///d:/College%20Work/Internship/BOM/Member%204/bin/).
2. Configuring default parameters and overrides inside [config.json](file:///d:/College%20Work/Internship/BOM/Member%204/config.json).
3. Executing the master pipeline [sbom_scanner.py](file:///d:/College%20Work/Internship/BOM/Member%204/sbom_scanner.py) to run the full scan-to-compliance workflow against target source code directories.
4. Monitoring validation output to ensure all 21 attributes are correctly populated and mapped to standard schemas.

## Architectural Workflow
```
[Target Source] 
       │
       ▼ (Step 1)
  Syft Scan (deep extraction) ──> [sbom_raw.json]
                                         │
                                         ▼ (Step 2)

                                                                PyPI Data Enrichment ──> [sbom_enriched.json]
                                                                                               │
                                                                                               ▼ (Step 4)
                                                                                      Compliance Validator
```

## Step-by-Step Execution Guide

### Step 1: Pre-requisites and Binary Setup
Deploy portable Windows x64 binaries locally to avoid requiring administrator privileges:
```powershell
# Run from workspace root
python "Member 4/setup_tools.py"
```
This automatically fetches:
- **Syft v1.3.0** (installed to `Member 4/bin/syft.exe`)
- **Grype v0.79.0** (installed to `Member 4/bin/grype.exe`)
And verifies their functionality via diagnostic commands.

### Step 2: Configure Attributes & Overrides
Update [config.json](file:///d:/College%20Work/Internship/BOM/Member%204/config.json) to set your organization's author info or package-specific overrides (like licensing notes, criticality tiers, or supplier details).

### Step 3: Execute the Primary Scanner Pipeline
Scan the target directory (e.g., [mock_project](file:///d:/College%20Work/Internship/BOM/mock_project)) and compile a fully validated compliance report:
```powershell
python "Member 4/sbom_scanner.py" --src "mock_project" --output "Member 4/sbom_final.json"
```

*What the scanner pipeline does internally:*
1. Runs `syft.exe dir:mock_project -o cyclonedx-json=Member 4/sbom_raw.json` to generate raw structural metadata.
2. Runs `grype.exe sbom:Member 4/sbom_raw.json -o cyclonedx-json --file Member 4/sbom_vulnerable.json` to inject vulnerability ratings (fulfilling **Attribute 8**).
3. Invokes the python enricher to resolve release dates, licenses, suppliers from PyPI, and custom metadata properties.
4. Invokes the validator to confirm that all 21 mandatory attributes are fully present and correctly formatted.

## 21-Attribute Mapping Matrix

| Attribute # | Attribute Name | CycloneDX Schema Mapping Location | Extraction & Resolution Method |
|---|---|---|---|
| **1** | Component Name | `components[].name` | Syft cataloger |
| **2** | Component Version | `components[].version` | Syft cataloger |
| **3** | Component Description | `components[].description` | PyPI JSON API / Overrides / Metadata |
| **4** | Component Supplier | `components[].supplier.name` | PyPI Author field / Config overrides |
| **5** | Component License | `components[].licenses[].license.name` | PyPI License field / Config overrides |
| **6** | Component Origin | Property: `origin` | Set as `open-source` (PyPI) or overridden |
| **7** | Component Dependencies | `dependencies` root array | Generated by Syft / Cleaned and padded by enricher |
| **8** | Vulnerabilities | `vulnerabilities` root array | Injected by Grype CLI scanner |
| **9** | Patch Status | Property: `patch_status` | Compares current version with PyPI's latest |
| **10** | Release Date | Property: `release_date` | Upload date fetched from PyPI API records |
| **11** | End-of-Life (EOL) Date | Property: `eol_date` | Calculated as `Release Date + offset` (default: 3 years) |
| **12** | Criticality | Property: `criticality` | Resolved via `config.json` overrides (default: `medium`) |
| **13** | Usage Restrictions | Property: `usage_restrictions` | Resolved via `config.json` overrides (default: `None`) |
| **14** | Checksums or Hashes | `components[].hashes` | SHA-256 hashes generated by Syft cataloger |
| **15** | Comments or Notes | Property: `comments` | Resolved via `config.json` overrides (default: `Scanned via Syft`) |
| **16** | Author of SBOM Data | `metadata.authors` | Configured inside `config.json` |
| **17** | Timestamp | `metadata.timestamp` | Generated dynamically at execution time (ISO 8601) |
| **18** | Executable Property | Property: `executable` | Set to `false` for libraries (overridable) |
| **19** | Archive Property | Property: `archive` | Set to `true` for PyPI packages (overridable) |
| **20** | Structured Property | Property: `structured` | Format descriptor: `CycloneDX JSON v1.6` |
| **21** | Unique Identifier | `components[].purl` | strictly formatted: `pkg:pypi/{name}@{version}` |
